# RETURN 홈페이지 운영가이드

운영진 인수인계용 문서. 서버를 처음 받았다면 위에서부터 순서대로.

## 1. 최초 서버 설정 (한 번만)

필요한 것: Python 3.11+, Node.js 20+, git

```bash
sudo useradd -m return
sudo git clone <저장소 URL> /srv/return-homepage
cd /srv/return-homepage/backend
python3 -m venv .venv && .venv/bin/pip install -r requirements.txt
cd ../frontend && npm install && npm run build
```

운영 환경변수 파일 `/srv/return-homepage/backend/.env.production` 작성:

```
RETURN_ADMIN_USERNAME=운영진아이디
RETURN_ADMIN_PASSWORD=강한비밀번호로교체
RETURN_SECRET_KEY=무작위긴문자열로교체
```

(`python3 -c "import secrets; print(secrets.token_hex(32))"` 로 SECRET_KEY 생성)

## 2. systemd 서비스 등록 (한 번만)

`/etc/systemd/system/return.service`:

```ini
[Unit]
Description=RETURN homepage
After=network.target

[Service]
User=return
WorkingDirectory=/srv/return-homepage/backend
EnvironmentFile=/srv/return-homepage/backend/.env.production
ExecStart=/srv/return-homepage/backend/.venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 80
AmbientCapabilities=CAP_NET_BIND_SERVICE
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now return
```

## 3. 평소 배포 (코드가 바뀔 때마다)

```bash
cd /srv/return-homepage && ./deploy.sh
```

## 4. 백업 (cron, 한 번만 등록)

```bash
mkdir -p /srv/return-homepage/backups
```

`crontab -e` (return 계정):

```
0 4 * * * cp /srv/return-homepage/backend/return.db /srv/return-homepage/backups/return-$(date +\%Y\%m\%d).db
```

복구: 백업 파일을 `backend/return.db`로 복사 후 `sudo systemctl restart return`

## 5. 자주 쓰는 명령

| 할 일 | 명령 |
|---|---|
| 서비스 상태 | `systemctl status return` |
| 로그 보기 | `journalctl -u return -f` |
| 재시작 | `sudo systemctl restart return` |
| 관리자 페이지 | `http://<서버주소>/admin` |
